# 宏组件系统介绍

## 📖 概述

宏组件是工作流系统中的一项强大功能，允许用户在节点的**指令**和**提示词**中使用特殊语法来动态引用其他节点的输出内容。这极大增强了工作流的灵活性和信息传递能力。

## 🚀 核心功能

### 1. 获取任意节点的全部内容

使用 `{{@节点名称}}` 语法可以获取指定节点的完整输出内容：

```
请参考之前的灵感输出：
{{@灵感采集}}
```

### 2. 获取特定类型的节点内容

使用 `{{@类型:节点类型}}` 语法可以获取指定类型节点的输出：

```
以下是之前生成的世界观设定：
{{@类型:worldview}}

角色档案如下：
{{@类型:characters}}
```

### 3. 获取前序节点列表

使用 `{{@前序节点}}` 语法可以获取当前节点之前所有已完成节点的摘要信息：

```
之前的创作进度：
{{@前序节点}}
```

### 4. 获取全局变量

使用 `{{@全局:变量名}}` 语法可以获取工作流全局上下文中的变量：

```
当前循环轮次：{{@全局:loop_index}}
当前分卷名称：{{@全局:current_volume_name}}
```

### 5. 获取小说数据

使用 `{{@小说:数据类型}}` 语法可以获取小说项目的基础数据：

```
小说标题：{{@小说:title}}
小说简介：{{@小说:description}}
总章节数：{{@小说:chapter_count}}
```

## 📋 支持的节点类型

| 类型键 | 类型名称 | 输出格式 |
|--------|----------|----------|
| `worldview` | 世界观设定 | 条目列表 |
| `characters` | 角色档案 | 角色列表 |
| `plotOutline` | 粗纲 | 大纲项列表 |
| `outline` | 大纲 | 大纲项列表 |
| `inspiration` | 灵感集 | 灵感项列表 |
| `aiChat` | AI对话 | 对话内容 |
| `userInput` | 用户输入 | 用户指令 |
| `saveToVolume` | 分卷规划 | 分卷内容 |
| `creationInfo` | 创作信息 | 分卷进度、循环轮次 |
| `chapter` | 章节 | 章节内容 |

## 🔧 使用场景示例

### 场景 1：在章节生成时引用世界观和角色

```
请根据以下设定生成章节内容：

## 世界观设定
{{@类型:worldview}}

## 角色档案
{{@类型:characters}}

## 粗纲内容
{{@粗纲生成}}

请按照大纲推进剧情...
```

### 场景 2：在分卷规划时引用灵感集

```
请根据以下灵感素材进行分卷规划：

{{@灵感采集}}

要求：
- 将灵感元素合理分配到各卷
- 每卷应有独立的主题和冲突线
- 共规划{{@全局:total_volumes}}卷
```

### 场景 3：在循环配置器中动态获取信息

```
当前创作状态：
- 小说名称：{{@小说:title}}
- 当前轮次：{{@全局:loop_index}}
- 总章节数：{{@小说:chapter_count}}

前序节点已完成内容：
{{@前序节点}}

请根据以上信息生成循环指令...
```

### 场景 4：跨节点内容传递

假设工作流中有多个节点：
1. 灵感采集节点（标签：灵感采集）
2. 世界观节点（标签：世界观设定）
3. 角色节点（标签：角色档案）
4. 粗纲节点（标签：粗纲生成）

在后续的大纲节点中：

```
## 创作依据

### 灵感来源
{{@灵感采集}}

### 设定基础
{{@世界观设定}}
{{@角色档案}}

### 结构参考
{{@粗纲生成}}
```

## 🎯 最佳实践

### 1. 始终使用节点标签

节点标签比节点ID更稳定，建议在宏中使用节点标签：

```
✅ 推荐：{{@世界观设定}}
❌ 不推荐：{{@node_abc123}}
```

### 2. 合理使用类型引用

当需要获取特定类型的所有节点输出时，使用类型引用：

```
{{@类型:worldview}}  // 获取所有世界观节点的输出
{{@类型:characters}} // 获取所有角色节点的输出
```

### 3. 注意引用顺序

宏只能引用**当前节点之前**已完成的节点内容。如果节点A在节点B之后，在节点A中引用 `{{@节点B}}` 将返回空内容。

### 4. 组合使用多个宏

可以在一个提示词中组合使用多个宏：

```
创作上下文：
{{@前序节点}}

小说基础信息：
{{@小说:title}} - {{@小说:description}}

当前状态：第{{@全局:loop_index}}轮创作
```

## 🔄 实现原理

宏组件系统在 `useWorkflowEngine.ts` 中实现：

1. **构建宏上下文** (`buildMacroContext`)：在每个节点执行前，收集前序节点信息、全局变量、小说数据等上下文。

2. **解析宏语法** (`resolveMacros`)：扫描文本中的 `{{@...}}` 语法，从上下文中提取对应内容进行替换。

3. **插值应用** (`interpolateWithMacros`)：在节点指令和提示词处理时自动应用宏解析。

相关文件：
- `src/components/Workflow/macros/index.ts` - 宏定义和解析逻辑
- `src/utils/WorkflowManager.ts` - `interpolateWithMacros` 方法
- `src/components/Workflow/hooks/useWorkflowEngine.ts` - 宏上下文构建

## ⚠️ 注意事项

1. **性能考虑**：大量使用宏可能导致上下文体积过大，影响AI响应速度。建议适度使用。

2. **内容截断**：如果节点输出内容过长，系统可能会进行截断以保持合理的上下文大小。

3. **空值处理**：如果引用的节点没有输出或不存在，宏将返回空字符串，不会中断工作流。

4. **特殊字符**：节点名称中避免使用特殊字符（如 `:`、`@`），以免影响宏解析。
