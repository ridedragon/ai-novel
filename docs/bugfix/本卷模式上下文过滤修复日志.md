# 本卷模式上下文过滤修复日志

## 修复日期
2026/4/7

## 问题描述

在本卷模式（`contextScope === 'volume'` 或 `'currentVolume'`）下，生成第二卷内容时，AI 请求中仍然包含了第一卷的内容，具体表现为：

1. **【小说知识库和参考】**显示了第一卷的世界观、角色等集合内容
2. **【前文回顾细节】**在新卷刚开始时不应该被发送（因为新卷没有前文）
3. **集合标注错误**：生成的第二卷世界观等内容，其集合名称标注仍然是第一卷的名称

## 问题根因分析

### 根因 1：集合注入没有按卷过滤

在 `autoInject` 函数中，当用户未手动选择集合时，代码会注入所有集合：

```javascript
// 旧代码
const autoInject = (selectedIds: string[], allSets: any[] | undefined) => {
  if (selectedIds && selectedIds.length > 0) {
    return resolvePending([...selectedIds], allSets);
  }
  // 问题：循环模式下会注入所有集合，包括其他卷的
  if (isLoopNode && allSets) {
    return allSets.map(s => s.id);
  }
  return [];
};
```

这导致第二卷生成时，第一卷的 `worldviewSets`、`characterSets` 等也被注入到 AI 请求中。

### 根因 2：前文回顾没有检查当前卷是否有章节

`summaryContextMessages` 的获取逻辑没有考虑本卷模式下新卷刚开始时没有章节的情况：

```javascript
// 旧代码
if (localNovel.chapters && localNovel.chapters.length > 0 && !isOutlineGen) {
  // 直接获取最后一章的总结，没有检查是否属于当前卷
  summaryContextMessages = getChapterContextMessages(localNovel, lastChapter, {...});
}
```

## 修复方案

### 修复 1：集合注入按卷过滤

在 `autoInject` 函数中增加本卷模式判断，只注入当前卷的集合：

```javascript
// 新代码
const isVolumeMode = globalConfig.contextScope === 'volume' || globalConfig.contextScope === 'currentVolume';
const currentVolumeName = currentWorkflowFolder || node.data.folderName || '';

const autoInject = (selectedIds: string[], allSets: any[] | undefined) => {
  // 如果已手动选择，则仅解析手动选择的
  if (selectedIds && selectedIds.length > 0) {
    return resolvePending([...selectedIds], allSets);
  }
  // 本卷模式修复：只注入当前卷的集合
  if (isVolumeMode && allSets && currentVolumeName) {
    const volumeSets = allSets.filter(s => s.name === currentVolumeName);
    terminal.log(`[CONTEXT] autoInject volume sets: ${volumeSets.length} for "${currentVolumeName}"`);
    return volumeSets.map(s => s.id);
  }
  // 如果是循环模式且未手动选择，默认全选
  if (isLoopNode && allSets) {
    return allSets.map(s => s.id);
  }
  return [];
};
```

### 修复 2：前文回顾按卷过滤

在本卷模式下，检查当前卷是否有章节，如果没有则不发送前文回顾：

```javascript
// 新代码
if (globalConfig.contextScope === 'volume' || globalConfig.contextScope === 'currentVolume') {
  effectiveVolumeId = (node.data.targetVolumeId as string) || workflowManager.getActiveVolumeAnchor() || '';
  
  // 本卷模式修复：如果当前节点所在卷没有章节（新卷刚开始），不发送前文回顾
  const volumeChapters = (localNovel.chapters || []).filter(c => c.volumeId === effectiveVolumeId);
  if (volumeChapters.length === 0) {
    terminal.log(`[CONTEXT] Volume mode: no chapters in current volume "${currentVolumeName}", skipping summary context`);
    summaryContextMessages = [];
  } else {
    // 使用当前卷的最后一章获取总结
    summaryContextMessages = getChapterContextMessages(
      localNovel,
      { ...volumeChapters[volumeChapters.length - 1], volumeId: effectiveVolumeId },
      {
        longTextMode: true,
        contextScope: 'volume',  // 本卷模式下只获取当前卷的总结
        contextChapterCount: globalConfig.contextChapterCount || 1,
      },
    );
  }
} else {
  // 整书模式：使用最后一章
  const lastChapter = localNovel.chapters[localNovel.chapters.length - 1];
  summaryContextMessages = getChapterContextMessages(localNovel, lastChapter, {...});
}
```

## 修复效果

修复后，本卷模式下的 AI 请求上下文将正确隔离：

| 上下文类型 | 修复前 | 修复后 |
|-----------|--------|--------|
| 【参考世界观】 | 所有卷的世界观 | 仅当前卷的世界观 |
| 【参考角色】 | 所有卷的角色 | 仅当前卷的角色 |
| 【参考粗纲】 | 所有卷的粗纲 | 仅当前卷的粗纲 |
| 【前文回顾】 | 最后一章的总结（可能是上一卷） | 当前卷的总结（新卷为空时不发送） |
| 集合名称标注 | 第一卷名称 | 当前卷名称 |

5. 检查新卷刚开始时是否没有发送前文回顾

## Bug 3: 分卷切换后 volumes 信息丢失

### 问题描述
在第二卷创作完成第三章后，第三卷生成的第一章内容跑到了第二卷里面，没有正确排到第三卷章节列表里面。

### 问题根因
在 `onChapterComplete` 回调中，分卷切换后 `localNovel = res` 会覆盖 `localNovel`，但 `res` 可能只包含 `chapters` 信息，不包含新创建的 `volumes` 信息。这导致后续章节生成时，`targetVolumeId` 指向的是旧的卷 ID。

### 修复方案
在 `onChapterComplete` 回调中，合并 `volumes` 信息，防止分卷切换后 volumes 丢失：

```javascript
// 旧代码
if (res?.chapters) localNovel = res;

// 新代码
if (res?.chapters) {
  // 核心修复：合并 volumes 信息，防止分卷切换后 volumes 丢失
  localNovel = {
    ...res,
    volumes: res.volumes && res.volumes.length > 0 ? res.volumes : localNovel.volumes,
  };
}
```

### 修复效果
修复后，分卷切换后 `localNovel.volumes` 信息不会丢失，后续章节会正确关联到新创建的卷。
