# 代码分析工作流卷节点运行模式调查文档

## 1. 引言

本文档详细分析从特定卷节点运行工作流的两个模式（单卷模式和全卷模式）的工作原理，以及生成章节放置的文件夹位置确定机制。所有结论均基于代码库中的实际代码实现。

## 2. 两个运行模式定义

从特定卷节点运行工作流有两个核心模式，定义在 [src/components/Workflow/types.ts](file:///workspace/src/components/Workflow/types.ts#L126-L132) 中：

```typescript
export type WorkflowRestartMode = 'volume' | 'full';

export interface WorkflowStartOptions {
  startIndex: number;
  targetVolumeId?: string;
  mode?: WorkflowRestartMode;
}
```

- `volume`: 单卷模式（仅清除当前卷内容）
- `full`: 全卷模式（清除当前卷及后续所有卷内容）

## 4. 单卷模式（volume）工作原理

### 4.1 清理范围

单卷模式仅清理用户指定的目标卷内容，不影响其他卷。关键逻辑在 [clearNovelContentByVolumes](file:///workspace/src/components/Workflow/hooks/useWorkflowEngine.ts#L199-L257) 函数中：

```typescript
const clearNovelContentByVolumes = (novel: Novel, volumeIds: string[], clearFollowingVolumes: boolean) => {
  if (!volumeIds.length) return novel;

  const volumeOrder = novel.volumes || [];
  const startOrderIndex = volumeOrder.findIndex(v => v.id === volumeIds[0]);
  const affectedVolumeIds = new Set(
    clearFollowingVolumes && startOrderIndex >= 0
      ? volumeOrder.slice(startOrderIndex).map(v => v.id)
      : volumeIds,
  );
  // ... 后续清理逻辑
}
```

当 `clearFollowingVolumes` 为 `false`（即单卷模式）时，`affectedVolumeIds` 只包含 `volumeIds` 中指定的卷。

### 4.2 单卷模式下的循环处理

在第 1314-1326 行，单卷模式下会跳过循环回跳，直接完成循环：

```typescript
// 核心修复：重写卷模式（单卷模式）下，不进行循环回跳
// 当用户指定了目标卷且有 mode（重写/全量模式），说明是"重写单卷"，只处理一卷
// 此时不应回跳到循环起点继续下一卷，而是在当前卷完成后直接完成循环
if (userSpecifiedTargetVolumeId && mode) {
  console.log('[LOOP_NODE] 单卷重写模式：跳过循环回跳，直接完成循环', {
    userSpecifiedTargetVolumeId,
    mode,
    currentLoopIndex,
    totalIterations: loopConfig.count,
  });
  terminal.log(`[LOOP_NODE] 单卷重写模式：当前卷已完成，不再回跳到下一卷`);
  // 标记循环完成
  await syncNodeStatus(node.id, { status: 'completed', loopConfig: { ...loopConfig, currentIndex: 0 } }, i);
  setEdgeAnimation(node.id, false);
  continue;
}
```

## 7. 工作流启动时的卷锁定机制

当用户从指定卷启动时，系统会在第 350-356 行设置启动卷锁：

```typescript
if (userSpecifiedTargetVolumeId && userSpecifiedTargetVolumeIndex >= 0) {
  workflowManager.lockStartVolume(userSpecifiedTargetVolumeId, userSpecifiedTargetVolumeIndex);
  workflowManager.setActiveVolumeAnchor(userSpecifiedTargetVolumeId);
  workflowManager.setCurrentVolumeIndex(userSpecifiedTargetVolumeIndex);
} else {
  workflowManager.clearStartVolumeLock();
}
```

此机制防止旧分卷规则错误地将执行回切到更早的卷（详细见 [WorkflowManager.ts](file:///workspace/src/utils/WorkflowManager.ts#L587-L608)）。

## 8. 总结

### 8.1 两个模式的核心区别

| 特性 | 单卷模式（volume） | 全卷模式（full） |
|------|-------------------|------------------|
| 清理范围 | 仅指定卷 | 指定卷及后续所有卷 |
| `clearFollowingVolumes` 参数 | `false` | `true` |
| 循环行为 | 完成当前卷后直接结束循环 | 正常循环处理所有卷 |

### 8.2 生成章节文件夹位置的确定优先级

1. **用户指定目标卷时**：`folderName` 直接设置为目标卷的 `title`（最高优先级）
2. **分卷规划节点（saveToVolume）**：使用第一个分卷的名称更新后续节点
3. **创建文件夹节点（createFolder/multiCreateFolder）**：使用节点指定的文件夹名称
4. **动态上下文扫描**：通过 `buildDynamicContext` 函数扫描前面节点确定
