### 2. 可能的问题原因

#### 2.1 API 配置问题
- **stream 参数未设置为 true**：在 API 请求中，`stream` 参数必须设置为 `true` 才能启用流式输出
- **模型不支持流式输出**：某些模型可能不支持流式输出，需要使用支持的模型如 gpt-3.5-turbo 或 gpt-4

#### 2.2 网络问题
- **网络延迟**：网络连接不稳定或延迟过高，导致流式数据无法及时到达
- **连接中断**：网络连接中断后重连，可能导致流式传输被重置

#### 2.3 前端处理问题
- **节流处理不当**：节流时间设置不合理，导致 UI 更新不及时
- **状态更新机制**：`setNovels` 中的合并逻辑可能有问题，导致流式更新被阻塞
- **组件渲染问题**：前端组件可能没有正确处理状态更新，导致界面不刷新

#### 2.4 代码逻辑问题
- **Delta Update 机制**：增量更新逻辑可能有问题，导致章节数据未正确合并
- **章节合并逻辑**：`setNovels` 中的流式更新检测逻辑可能不准确
- **异步处理**：异步操作顺序问题，导致状态更新被覆盖

## 修复过程

### 问题定位

通过代码分析，发现了导致流式输出丢失的关键问题：在 `useWorkflowEngine.ts` 中，流式传输循环内部调用了 `await updateLocalAndGlobal(localNovel);` 异步函数，这会导致流式循环暂停，从而失去流式效果。

### 问题原因

`updateLocalAndGlobal` 函数包含 `await new Promise(resolve => setTimeout(resolve, 0));` 语句，这会导致 JavaScript 事件循环切换，中断流式数据的连续处理，使得所有内容在 AI 生成完成后才一次性显示。

### 修复方案

修改 `useWorkflowEngine.ts` 文件中的流式传输循环，将异步的 `updateLocalAndGlobal` 调用替换为同步的状态更新逻辑，避免阻塞流式循环：

1. **移除异步调用**：删除 `await updateLocalAndGlobal(localNovel);`
2. **实现同步更新**：直接在流式循环中执行状态更新逻辑
3. **保持功能完整**：确保状态更新的所有功能都得到保留

### 修复代码

在 `src/components/Workflow/hooks/useWorkflowEngine.ts` 文件中，修改了以下代码：

```typescript
// 修复前
await updateLocalAndGlobal(localNovel);

// 修复后
const volumeStateMap = new Map();
(activeNovelRef.current?.volumes || []).forEach(v => volumeStateMap.set(v.id, v.collapsed));

const mergedNovel: Novel = {
  ...localNovel,
  volumes: (localNovel.volumes || []).map(v => ({
    ...v,
    collapsed: volumeStateMap.has(v.id) ? volumeStateMap.get(v.id) : v.collapsed,
  })),
};
localNovel = mergedNovel;

if (onUpdateNovel) {
  onUpdateNovel(mergedNovel);
}
```
