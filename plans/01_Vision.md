# AI 小说生成器项目计划书 - 第一部分：项目愿景与背景

## 1.1 项目背景：从“辅助工具”到“协同创作伙伴”

在过去的一年里，大语言模型（LLM）彻底改变了内容创作的版图。然而，目前的 AI 小说创作工具大多停留在“单点交互”层面——即用户输入一段提示词，AI 返回一段文字。这种模式在创作短篇故事或简单段落时表现尚可，但在处理百万字规模的长篇小说时，往往会面临以下致命问题：

1. **逻辑断层与“吃书”现象**：由于上下文窗口限制，AI 容易忘记前文设定的角色性格、世界观背景或已发生的关键剧情。
2. **缺乏宏观控制**：AI 难以把握小说的整体节奏、伏笔铺设以及高潮起伏，导致叙事平铺直叙，缺乏文学美感。
3. **人设崩塌**：角色在不同章节的行为逻辑可能出现前后矛盾，缺乏跨章节的成长弧光。
4. **创作灵感的碎片化**：灵感、设定、大纲和正文散落在不同地方，缺乏一个有机的系统将它们串联起来。

本项目旨在设计并实现一个**基于多 Agent（智能体）协同的 AI 小说生成器**。它不再仅仅是一个简单的接口封装，而是一个具备“自我管理”、“逻辑审计”和“深度记忆”能力的智能创作站。

## 1.2 产品愿景：打造“全自动编剧工作室”

我们的愿景是为每一位创作者提供一个专属的“全自动编剧工作室”。在这个工作室里，创作者扮演的是“总导演”或“主编”的角色，而具体的细节填充、设定维护、逻辑质检则由一组各司其职的 Agent 协同完成。

### 核心价值主张

* **极致的一致性**：通过专门的设定 Agent 和知识图谱技术，确保百万字小说无逻辑矛盾。
* **文学级叙事**：引入剧情架构 Agent，模拟好莱坞剧作法或网文黄金三章结构，提升叙事张力。
* **深度定制化**：用户可以深度定制每一个 Agent 的“灵魂”，包括文风、偏好、甚至逻辑严苛度。
* **低门槛高上限**：既能为小白用户一键生成大纲和正文，也能为专业作家提供精准的段落优化和逻辑对齐。

## 1.3 市场与用户分析

### 1.3.1 目标用户画像

1. **网络小说作家**：需要快速产出大量文字，但又希望保持高质量的情节逻辑和文笔。
2. **剧本杀/游戏策划**：需要构建复杂的多分支剧情和庞大的世界观体系。
3. **内容创作爱好者**：有脑洞、有创意，但缺乏长篇写作经验和文字驾驭能力的普通用户。
4. **同人创作群体**：需要在既定角色框架下进行高一致性的续写或改编。

### 1.3.2 行业现状与竞品差异

目前市面上的竞品（如 Sudowrite, NovelAI 等）虽然在文笔润色和片段生成上表现出色，但在**多 Agent 自动协同**和**长效逻辑审计**方面仍有较大提升空间。

本项目的差异化优势在于：

* **原生多 Agent 架构**：从底层设计上支持 Agent 间的对话和博弈（例如：创作 Agent 写完后，审计 Agent 会提出修改意见并要求重写）。
* **深度集成与可扩展性**：基于 [`src/App.tsx`](src/App.tsx) 已有的扎实基础，快速扩展复杂的业务逻辑。

## 1.4 技术演进路径设计

我们将项目分为三个阶段实施：

1. **阶段一：基础设施与单 Agent 增强（当前状态）**
    * 完善 [`src/components/CharacterManager.tsx`](src/components/CharacterManager.tsx) 和 [`src/components/WorldviewManager.tsx`](src/components/WorldviewManager.tsx)。
    * 建立基础的长文总结机制 [`src/utils/SummaryManager.ts`](src/utils/SummaryManager.ts)。

2. **阶段二：多 Agent 协同系统（本项目重点规划）**
    * 实现“导演 Agent”调度逻辑。
    * 建立基于向量数据库的长效记忆系统（RAG）。
    * 设计 Agent 间的通讯协议和冲突仲裁机制。

3. **阶段三：生态与个性化（未来愿景）**
    * 支持 Agent 插件化，用户可以分享自己的“毒舌评论 Agent”或“华丽文风 Agent”。
    * 引入多模态 Agent，自动为小说章节生成配图或音乐。
