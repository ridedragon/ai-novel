# 手机端编辑交互说明

## 背景

由于手机端的选择模式使用体验不佳，我们重新设计了手机端的编辑交互逻辑，通过标记方式来替代传统的选择方式。

## 新交互方式

### 标记模式

在手机端编辑界面中，用户可以通过输入特定格式来标记需要修改的内容：

```
【*需要修改的内容*】
```

例如：
```
【*他慢慢地走在路上，心里想着今天发生的事情。*】
```

### 操作步骤

1. **进入编辑模式**：点击编辑按钮进入编辑模式
2. **标记需要修改的内容**：在文本中使用 `【*` 和 `*】` 包裹需要修改的部分
3. **输入修改要求**：在下方的输入框中输入具体的修改要求
4. **发送修改请求**：点击发送按钮，AI会根据标记的内容和修改要求进行处理
5. **查看修改结果**：修改完成后，标记会被替换为修改后的内容

### 示例

1. 原始文本：
   ```
   她站在窗前，看着外面的雨。
   ```

2. 标记需要修改的内容：
   ```
   【*她站在窗前，看着外面的雨。*】
   ```

3. 输入修改要求：
   ```
   增加一些环境描写，让场景更有画面感
   ```

4. 点击发送按钮，AI处理后：
   ```
   她伫立在落地窗前，指尖轻轻抵着微凉的玻璃，凝视着窗外细密的雨丝。雨幕模糊了远处的街灯，在玻璃上晕开一片朦胧的光晕，空气中弥漫着湿润的泥土气息。
   ```

## 技术实现

### 设备检测

系统会自动检测设备类型，当屏幕宽度小于768px时，自动切换到手机端模式。

### 标记解析

系统会实时解析文本中的标记格式 `【*内容*】`，并在界面下方显示已标记的内容。

### AI处理

当用户发送修改请求时，系统会：
1. 提取所有标记的内容
2. 构建包含标记内容和修改要求的请求
3. 发送给AI模型进行处理
4. 接收AI返回的结果并应用到文本中

### 电脑端兼容

电脑端保持原有的选择模式，用户可以通过鼠标选择文本进行修改，确保不同设备的用户都能获得良好的使用体验。

## 注意事项

1. 标记格式必须严格按照 `【*内容*】` 的格式，否则系统无法识别
2. 可以同时标记多个部分，系统会一次性处理所有标记的内容
3. 标记的内容不宜过长，建议每个标记不超过200字
4. 修改完成后，标记会被自动移除，替换为修改后的内容

## 优势

1. **操作简单**：不需要在手机屏幕上进行复杂的选择操作
2. **精确控制**：用户可以精确标记需要修改的内容
3. **多段处理**：可以同时标记多个部分进行批量处理
4. **直观明了**：标记的内容会在界面下方显示，方便用户确认

## 常见问题

### Q: 标记后没有显示在下方怎么办？
A: 请检查标记格式是否正确，确保使用了 `【*` 和 `*】` 包裹内容。

### Q: 标记的内容被AI修改后，标记符号会保留吗？
A: 不会，修改完成后标记符号会被自动移除，只保留修改后的内容。

### Q: 可以同时标记多少个部分？
A: 理论上没有限制，但为了确保处理效果，建议同时标记不超过5个部分。

### Q: 电脑端是否可以使用标记模式？
A: 电脑端默认使用选择模式，但如果需要，也可以手动使用标记格式进行编辑。
